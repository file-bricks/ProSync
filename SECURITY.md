# Security Policy / Sicherheitsrichtlinie

## Deutsch

### Unterstützte Versionen

| Version | Unterstützt |

### Sicherheitslücken melden

Wenn Sie eine Sicherheitslücke finden, melden Sie sie bitte über GitHubs private Vulnerability-Reporting-Funktion:

1. Öffnen Sie den Tab **Security** in diesem Repository
2. Wählen Sie **Report a vulnerability**
3. Beschreiben Sie die Schwachstelle, Reproduktion und mögliche Auswirkungen

Bitte öffnen Sie für Sicherheitslücken keine öffentlichen Issues und veröffentlichen Sie keine privaten Pfade, Konfigurationen oder personenbezogenen Daten.

Berichte werden nach bestem Aufwand geprüft. Eine feste Antwortzeit gibt es nicht, bestätigte Sicherheitskorrekturen haben aber Vorrang vor regulären Features.

## English

### Supported Versions

| Version | Supported |
| ------- | --------- |
| latest  | yes       |

### Reporting a Vulnerability

If you discover a security vulnerability, please report it via GitHub's private vulnerability reporting feature:

1. Go to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Provide reproduction details and potential impact

Please do not open public issues for security vulnerabilities, and do not publish private paths, configuration data, or personal data.

Reports are reviewed on a best-effort basis. There is no guaranteed response time, but confirmed security fixes will be prioritized over regular feature work.
